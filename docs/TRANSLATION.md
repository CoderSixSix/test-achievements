# Translation Guide

This guide helps translators contribute to internationalizing the GitHub Achievements Tracker.

## Translation Process

### 1. Choose a Language
See our list of priority languages in the i18n support issue.

### 2. Get Started
1. Fork the repository
2. Create a new branch: `git checkout -b translation/your-language`
3. Locate the files to translate
4. Make your translations
5. Submit a pull request

### 3. Files to Translate

#### Documentation Files
- README.md
- CONTRIBUTING.md
- SECURITY.md
- CODE_OF_CONDUCT.md

## Quality Standards
- Use natural, fluent language
- Maintain technical accuracy
- Keep formatting consistent
- Preserve code examples
- Test in context

## Recognition

Translators will be:
- Listed in contributors
- Mentioned in release notes
- Credited in documentation
